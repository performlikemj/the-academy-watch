## Key API assumptions (all verified against live source/docs this session)
- **rfdetr==1.7.1**: `RFDETRNano/Small/Medium/Large` (`RFDETRBase` deprecated; XL/2XL are PML-licensed — excluded); constructor accepts `device=` and `resolution=`; `model.predict(images, threshold=0.5, include_source_image=False)` accepts np.ndarray RGB single-or-list, returns `sv.Detections` or list; **pretrained COCO checkpoints emit raw sparse COCO category IDs** (person=1, sports ball=37) with `detections.data["class_name"]` available — I filter by name with ID fallback; `optimize_for_inference(compile=True, batch_size=N)` exists (torch.jit.trace) — opt-in only because the fixed-batch trace conflicts with slicer single-tile calls
- **trackers==2.4.0**: `from trackers import BoTSORTTracker`; `BoTSORTTracker(frame_rate=12.5, lost_track_buffer=30, enable_cmc=True)`; `update(detections: sv.Detections, frame: np.ndarray|None) -> sv.Detections` with tracker_id (−1 = unconfirmed, filtered out) — read directly from `src/trackers/core/botsort/tracker.py@develop`
- **supervision==0.28.0**: `InferenceSlicer(callback, slice_wh=640, overlap_wh=100)` called as `slicer(image)` (current param is `overlap_wh`, not the old `overlap_ratio_wh`); `EllipseAnnotator`/`LabelAnnotator` with `ColorLookup.TRACK`
- **transformers==5.10.2**: `SiglipVisionModel.from_pretrained("google/siglip-base-patch16-224")` + `AutoProcessor`; embeddings from `outputs.pooler_output` (verified present in v5.10.2 docs); weights Apache-2.0
